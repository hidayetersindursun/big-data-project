🕳️
Systematic Data Void
No unified, machine-readable pipeline tracks the farm-to-shelf journey. The supply chain is a completely unmeasured black box — regulators and citizens have no real-time data to act on.
🚀
Asymmetric Price Pass-Through ("Rockets & Feathers")
When input costs rise, retail prices rocket up instantly. When costs fall, prices feather down slowly — or not at all. This asymmetry is a structural consumer harm never proven with data.
📍
Location-Based Speculation
Same wholesale entry price → drastically different district-level retail prices. Istanbul Kadıköy: ₺35 vs Izmir Karşıyaka: ₺25 for the same product from the same hall. No accountability mechanism exists.
⚡
Unmeasured Supply Shock Propagation
When a frost hits Antalya, how many days until Istanbul's retail shelves reflect the shock? When energy prices rise, how much of the logistics cost increase passes through to the consumer? These chains are invisible.

 Pipeline Architecture

Data Journey: From Source to Insight

Real-time streams (Kafka) and scheduled batch jobs (Airflow) feed raw data into the Bronze layer. Apache Flink & Spark handle entity resolution and cleansing in the Silver layer. Aggregated margin and shock-propagation metrics are served from the Gold layer via Trino → Superset.

 Data Engineering

Three Hard Problems We Solve

🔗

Challenge 1 · Entity Resolution

Hall API: "Domates Sofralık Sera"
Retail slug: "salkim-domates-1-kg"

No shared ID exists between systems. Solution: TF-IDF + Cosine Similarity NLP pipeline via Spark MLlib matches products across schemas with no manual mapping.

⚖️

Challenge 2 · Unit Standardization

Hall pricing: Kasa / Çuval / Bağ
Retail pricing: Gram / Adet / Paket

Solution: Custom Spark UDFs convert all units to 1 KG equivalent before any price comparison or aggregation.

🕐

Challenge 3 · Temporal & Spatial Alignment

GDELT: 15-min · EPIAS: hourly
Hall: daily · Tonnage: monthly

Solution: Delta Lake As-of Join aligns mismatched time granularities. Spatial Join maps each province's retail data to its nearest wholesale hall.

Stream Processing
Apache Flink for real-time entity resolution, unit conversion, and anomaly flagging on the incoming Kafka stream — before data hits Bronze.
Batch Transformation
Spark Structured Streaming + Airflow-triggered jobs rebuild Silver and Gold layers nightly, incorporating all newly ingested batch data from halls and CBRT.
Anomaly Detection
Isolation Forest on the Silver stream flags when hall prices hold steady but retail price spikes ≥50% — distinguishing data errors from genuine speculative pricing events.
